# План эволюции сервиса: от Python-скрипта к автономному ИИ-агенту

## 1. Концептуальный переход
Текущий проект (линейный скрипт) преобразуется в **Агентскую систему**. 
*   **Было:** Скрипт вызывает API, получает текст и шлет его по SMTP.
*   **Стало:** Агент получает цель, сам планирует шаги, обращается к истории клиента, генерирует контент (текст + фото) и управляет жизненным циклом поздравления через систему статусов.

## 2. Гибридная архитектура данных
Для работы агента внедряется двухслойная система баз данных:
*   **Реляционная БД (PostgreSQL/MySQL):** "Источник истины". Хранит структурированные данные: ФИО, даты рождения, балансы, часовые пояса и статусы VIP. Используется для точного срабатывания триггеров.
*   **Векторная БД (ChromaDB/FAISS):** "Память и опыт". Хранит семантические векторы прошлых переписок. Позволяет агенту "вспоминать" контекст: о чем общались ранее, какие хобби упоминались, на что клиент жаловался.

## 3. Технологический стек (Future Stack)
*   **Мозг:** GigaChat Pro/Max (через API).
*   **Фреймворк:** GigaChain (адаптация LangChain для работы с инструментами и памятью).
*   **Генерация образов:** Kandinsky API (создание уникальных открыток по промпту агента).
*   **Бэкенд:** FastAPI (перевод логики на REST API для обмена JSON-данными).
*   **Фронтенд:** React + Tailwind CSS (современная интерактивная админ-панель).

## 4. Автономный конвейер и Очередь
*   **Ежедневный запуск (Cron 00:00):** Python-скрипт выбирает события дня из SQL и создает очередь задач.
*   **Агентский цикл:** Для каждой задачи агент подтягивает контекст из векторной БД -> пишет текст -> генерирует промпт для картинки -> проходит этап самопроверки (Self-Reflection).
*   **Умная доставка:** Сообщения сохраняются в статусе `SCHEDULED` и отправляются через **Email API** (вместо сырого SMTP) строго в 10:00 по местному времени клиента.

## 5. Панель управления (Human-in-the-Loop)
Вместо устаревших Jinja2-шаблонов внедряется React-интерфейс для сотрудников:
*   **Мониторинг:** Live-лента работы агента и статистика (отправлено/прочитано).
*   **VIP-контроль:** Автоматическая постановка поздравлений VIP-клиентов в статус `AWAITING_REVIEW`. Агент не отправит письмо, пока сотрудник не нажмет "Одобрить" или не поправит текст вручную.
*   **Редактор:** Возможность перегенерировать картинку или текст одним кликом, если результат агента не устроил сотрудника.

## 6. Логика статусов поздравления
1. `PENDING` — задача создана планировщиком.
2. `PROCESSING` — агент работает над контентом.
3. `AWAITING_REVIEW` — (для VIP) ожидает проверки человеком.
4. `SCHEDULED` — контент готов и ждет времени отправки.
5. `SENT` — успешно отправлено через API.
6. `ERROR` — зафиксирована ошибка (логи сохраняются для обучения агента).
