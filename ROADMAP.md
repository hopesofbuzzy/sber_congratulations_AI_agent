# ROADMAP (команда 5 человек, хакатон → MVP → развитие)

Ниже — последовательный план работ, чтобы за хакатон получить демонстрируемый **MVP конвейера**, а затем эволюционировать в «боевое» решение.

## Роли (условно 5 участников)

1. **Backend/DB инженер** — API, модели данных, миграции, бизнес-логика статусов.
2. **AI/Agent инженер** — оркестратор, шаблоны/тон, провайдеры LLM, guardrails, идемпотентность генерации.
3. **Web/UX инженер** — веб-интерфейс, сценарии ручного управления, таблицы/фильтры/статусы.
4. **Data/Enrichment инженер** — календарь праздников, справочники, обогащение по ИНН (ОКВЭД/руководитель), кэш.
5. **DevOps/QA инженер** — локальный запуск, CI, тесты, наблюдаемость, документация, контроль качества.

## Что уже можно показывать на презентации

### Ключевые результаты текущего этапа

1. **Backend/DB**
   - Расширена модель `Client`: `ИНН`, официальное название компании, `ОКВЭД`, руководитель, сайт, `source_url`, статусы enrichment.
   - Добавлены bulk/manual endpoints и web-actions для enrichment профилей компаний.
   - Сохранён безопасный SQLite-upgrade через lightweight migration без удаления локальной БД.
   - Добавлены режимы управления доставкой через `.env`: отправка сразу или строго в дату события.
2. **AI/Agent**
   - Генерация текста теперь использует более богатый FACTS-контекст: отрасль (`okved_name`) и официальное название компании.
   - Обновлены fallback-генерация и LLM-промпты, чтобы персонализация опиралась на реальные поля организации.
   - Заложен feedback-loop как источник будущего улучшения качества и ранжирования поздравлений.
   - Контур доставки и approve-сценарий синхронизированы с конфигурацией `event_date/immediate`.
3. **Web/UX**
   - На странице `Clients` появился управляемый сценарий `Seed -> Enrich company profiles -> Run agent`.
   - В UI показываются enrichment-статусы, `ИНН`, `ОКВЭД`, руководитель и ошибки enrichment.
   - На странице `Greetings` добавлен inline feedback менеджера и расширен дашборд метриками.
   - Выполнен presentation-ready UI refresh: единый визуальный стиль, улучшенная навигация, hero-блоки, KPI-карточки и более сильная подача сценария демо.
   - Доработаны reset-сценарий, импорт клиентской базы, компактное отображение контактов и UX актуализации через внешний источник.
4. **Data/Enrichment**
   - Локальный справочник и CSV-база компаний вынесены в выделенную директорию ресурсов; импорт теперь формирует готовые клиентские профили.
   - Реализован провайдерный enrichment-контур `demo / dadata / hybrid` с возможностью внешней актуализации профилей по ИНН.
   - Seed demo data теперь создаёт клиентов с `ИНН` и готовностью к enrichment; импорт реальной базы заполняет базовые `segment/profession`.
5. **DevOps/QA**
   - Добавлены новые тесты на enrichment, CSV-импорт, reset demo-state, режимы доставки, HTML-страницы и использование enriched-данных в генерации.
   - Поднят локальный `.venv` и прогнан полный backend test suite.
   - Актуализированы `README.md`, `AI_HANDOFF.md`, `docs/DECISIONS.md` и `env.example`.

### Демонстрационный сценарий

1. `Seed demo data`
2. `Enrich company profiles` или `Импортировать базу компаний`
3. При необходимости `Актуализировать через внешний источник`
4. `Run agent now`
5. Показать, что в `Clients` появились `ИНН/отрасль/руководитель`, а контакты отображаются компактно
6. Показать персонализированные `Greetings`
7. Сохранить feedback менеджера
8. Показать `Deliveries` и outbox-артефакты

## Текущая проектная конфигурация

- **Контур данных**: CRM-профиль клиента + организационные enrichment-поля (`ИНН`, `ОКВЭД`, руководитель, источник).
- **Контур источников**: локальный CSV/registry для загрузки и внешний провайдер `DaData` для актуализации по ИНН.
- **Контур AI**: rule-based fallback + optional LLM, персонализация на основе enriched-профиля компании.
- **Контур оператора**: ручной review для VIP, feedback менеджера, аудит запусков агента.
- **Контур демонстрации**: presentation-ready UI, управляемый сценарий demo flow, воспроизводимый seed данных, полная очистка demo-state.

## Этап 0 — выравнивание (0.5 дня)

- **Общее**
  - Согласовать цели демо: какие сценарии покажем (ДР, праздник, ручной триггер, доставка).
  - Определить минимальный набор полей «CRM» для персонализации.
  - Зафиксировать «канал доставки» для MVP (файловый outbox) и формат.

**Выход**: согласованный сценарий демо + список сущностей/полей.

## Этап 1 — Каркас MVP (день 1)

- **Backend/DB**
  - Поднять FastAPI, SQLite, модели: Client/Event/Greeting/Delivery/Holiday/Feedback.
  - Реализовать CRUD клиентов и список событий/поздравлений.
- **Web/UX**
  - Дашборд + страницы: Clients, Events, Greetings/Deliveries, Settings (Holidays).
  - Кнопки: seed demo data, run agent now, manual trigger.
- **DevOps/QA**
  - Скрипты запуска на Windows, `env.example`, базовая структура тестов.

**Выход**: UI показывает данные, всё запускается локально.

## Этап 2 — Конвейер агента (день 2)

- **AI/Agent**
  - Детектор событий на \(N\) дней вперёд (ДР + праздники из справочника).
  - Выбор тона/шаблона по сегменту и типу события.
  - Генерация текста: LLM-провайдер (опционально) + «безключевой» rule-based fallback.
  - Генерация «открытки»: простой рендер (Pillow) в файл.
  - Логи и статусы, идемпотентность на повторных запусках.
- **Backend/DB**
  - Эндпоинт «run agent once», «trigger manual event», фиксация Delivery.
- **Web/UX**
  - Отображение прогресса: статусы, фильтры «к отправке/отправлено/ошибка».

**Выход**: кнопка «Run agent now» создаёт поздравления и «отправляет» их в outbox.

## Этап 3 — Обогащение данных (день 3, если успеваем)

- **Data/Enrichment**
  - Справочник профессиональных праздников (источник + формат + обновление).
  - MVP-обогащение по ИНН: получить ОКВЭД/руководителя (через публичные источники/скрапинг/заглушки).
  - Кэширование и ограничения частоты запросов.
- **AI/Agent**
  - Использовать ОКВЭД/отрасль для более точных пожеланий.

**Выход**: персонализация учитывает отрасль и название компании.

## Этап 4 — Надёжность и качество (параллельно, непрерывно)

- **DevOps/QA**
  - Тесты: детектор событий, идемпотентная отправка, базовые API.
  - Статические проверки: форматирование/линт (ruff/black), типы (mypy) — по возможности.
  - Наблюдаемость: структурные логи, трассировка ошибок, отчёт по запуску агента.
- **Backend/AI**
  - Ретраи сетевых вызовов, таймауты, ограничение параллелизма.
  - Guardrails текста: запрет персональных данных сверх разрешённых, анти-«галлюцинации».

## Этап 5 — Пост-MVP развитие (после хакатона)

- **Каналы доставки**
  - SMTP/Exchange, SMS, WhatsApp/Telegram (через корпоративные шлюзы), контроль согласий.
- **Интеграция с CRM**
  - Read-only выгрузка, CDC/ETL, аудит изменений, роли/ACL.
- **Обучение на результате**
  - Метрики: открываемость, ответы, NPS/оценка менеджера.
  - A/B тесты шаблонов, обучение ранжирования/политики выбора тона.
- **Комплаенс/безопасность**
  - Политики хранения данных, маскирование, журналирование доступа, DLP.
